# 04/03炉资源必要资源量与工作分割

炉资源可以放100个中间品，则必要资源量设为0.01（做一个产品占用炉的资源量的0.01）。假设炉资源的必要资源量为0.01，当有3个订单，每个订单的制造数量为50，且这三个订单加工条件相同，则需要将3个订单进行工作分割成150份，然后由于炉资源的必要资源量为0.01，即一次可以处理2个订单的100份工作，所以一次性可以排两个订单进去；若不进行工作分割，则炉资源会按订单计算，以为此时有3/100个任务，于是就将3个订单一次性排进一趟炉资源处理中，而这是不对的。若觉得订单分割较为麻烦，可在资源表中对炉资源的`分派资源量标志`设置为`与制造数量成比例`，此时炉资源就会自动识别订单中的制造数量，以制造数量为单位进行分派，而不是以订单数量为单位进行分派。

# 04/03静态分割与动态分割

- 静态分割

工作表中，系统必须先正常展开一次，把工序、物料、中间品都算出来，只有生成了工作，才可以编辑分割参数，分割参数编辑好后，要想使其生效，必须再展开一次，使系统按照编辑好的分割参数进行排程。（最上位收集条件：分割源工作以外【只显示实际工作，不显示工作根】）。

品目表中对最终品目或中间品进行分割，不需要二次展开，但是对中间品的分割要注意，只有展开一次后才会显示中间品，此时才能对中间品进行分割。

- 动态分割

资源表中，即便对机器设置了资源分割数量，排程没有用到该机器时，也不会生成相应的分割结构，只有这台机器真正被排程用到时，系统才会生成对应的分割结构，所以是动态分割。制造BOM表也同理，只有这个品目 / 工序真正被使用时，才会生成对应的分割结构。

计划参数表中的允许分割属性在展开和排产时生效，此时即便表中设置了分割，优先级也是取决于计划参数表，同时还要注意计划参数表的分割工作是否靠拢在一起。在工艺bom中只显示使用指令，在物料bom中只显示输入指令。

在订单展开时订单下面挂的是工作，工作上挂的输入输出指令（未排程，资源不确定是否使用，故没有使用指令），排产后发生变化，工作下面挂任务（前设置/制造/后设置），任务下面挂输入输出指令。原料数量在输入指令中的前设置/制造/后设置都可以填，填在哪个参数下，排程时指令就挂在哪个任务下。

# 04/10BOM分类

工艺BOM中的字段通常要加在使用指令模板中，物料BOM的字段通常加在输入指令模板中，而均有的字段则通常加在所有指令模板中。这是由于某些与指令相关的字段在从其他字段取值时只能在该指令模板所拥有的字段中取到，例如，制造字段在输入、输出、使用指令模板中均有，但是如果制造字段属于输入指令模板，并且要从其他字段取值时，只能从输入指令模板中拥有的字段中取值，而无法从其他指令模板中取到。

## 工艺BOM分类

![image-20260410145803339](https://img.tynote.cn/img/typora/20260410145812567.png#800w)

### 标准工艺BOM

定义：产品分类维度的通用工艺模板，不绑定具体物料，只定义<font color='blue'>某类产品</font>的标准工艺规则。

作用：为料号级 BOM 提供通用工艺模板，避免重复维护。

标准工艺BOM两种生成方式：①按照制造BOM表结构手工填入②手工维护分表，最终将分表拼接成标准工艺BOM

分表有6张：工艺BOM-工艺路线、工艺BOM-与主资源对应、工艺BOM-与副资源对应、工艺BOM-主资源与副资源对应、工艺BOM-工时、工艺BOM-复杂工艺。

注：维护工艺BOM-工时表时，若同一工序的多个主资源所需工时相同，也就是制造字段的值相同，换句话说，也就是工序所需工时与资源无关，那么此时可以省略主资源字段的值，将同一工序的多条工时相同的记录合并为一条记录，提高该分表的维护效率。

### 料号级工艺BOM

定义：绑定具体物料（料号）的**单品工艺标准**，是连接标准与订单的核心中间层。

生成方式：①从标准类型工艺 BOM 生成（可直接复用通用模板，快速生成单品工艺）②手工维护分表，最终将分表拼接成料号级工艺BOM③直接手动输入

作用：为订单级 BOM 提供**单品工艺依据**

### 订单级工艺BOM

定义: 一定是当前时间内确定的要进行生产的订单。绑定具体生产订单的**个性化工艺**，是 APS 排程、车间现场执行的最终数据。

注：如果是未来一段时间内未开出制造订单的，我们称为计划订单。

生成方式：①从料号级工艺 BOM 生成（可直接复用通用模板，快速生成单品工艺）②直接手动输入

### 标准工艺BOM与订单工艺BOM的制造字段的取值区别

在HANSAFLEX项目中，有四个BOM表（实际上是一张底表，按照不同的业务场景类型筛选并显示不同的数据而已）。

不同BOM表中的制造字段在排产时的区别：

在标准工艺BOM中，制造字段的意思通常就是单位需求量，即该工艺路线生产一个最终品目所需要的原料或者某个中间品目的数量，所以在导入客户订单分派工作时，实际的原料或中间品的需求量为`制造字段的值×订单数量`

但在其订单工艺BOM表中，现有两个字段分别是需求数量和已领数量，所以实际需要生产的数量是会发生变化的，实际需要生产的数量=需求数量-已领数量。订单工艺BOM本质上是BOM表，但由于一些字段的原因，在用户视角更具有订单属性，同时由于1️⃣`需求数量字段-已领数量=未领数量`已经给出了该道工序所需要的原料或中间品目的数量，2️⃣在BOM中计算实际的原料或中间品目的需求量时会拿制造字段的值×订单数量。而我们此时仅靠未领数量就可以计算得出，不再需要制造字段的值去乘订单数量，所以需要在其乘之前将其抵消，所以制造字段的值应该修改为`（需求数量-已领数量）/订单数量`，也就是`未领数量/订单数量`，这样程序在计算实际需要生产的数量时的步骤是`制造字段的值×订单数量=（需求数量-已领数量）/订单数量×订单数量=需求数量-已领数量=未领数量`

2. AdvanceAlongResourceWorkingDay(对象;日期;整数;时间间隔;整数)：从指定的时间开始，按照【资源工作日历】往后(前)推进  N 个工作日，返回最终的时间。/返回沿着资源稼动时间过所指定天数之后的时刻。
